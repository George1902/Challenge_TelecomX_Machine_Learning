# Challenge_TelecomX_Machine_Learning
Tercer Challenge de la formación en data science de Alura Latam con python

📊 Telecom X — Parte 2: Predicción de Cancelación (Churn)📝 Descripción del ProyectoTras el éxito del Análisis Exploratorio de Datos (EDA) en la Fase 1, este proyecto avanza hacia la inteligencia predictiva. El objetivo es construir un sistema de aprendizaje automático capaz de identificar a los clientes con alta probabilidad de cancelar sus servicios en Telecom X.Esta solución permite al equipo de marketing actuar de forma proactiva, aplicando estrategias de retención personalizadas antes de que el cliente abandone la compañía.🎯 Objetivos TécnicosPipeline de Datos: Limpieza, codificación (One-Hot Encoding) y estandarización.Balanceo de Clases: Implementación de técnicas para manejar el desequilibrio de datos (SMOTE).Modelado: Comparación de algoritmos lineales (Regresión Logística) y basados en árboles (Random Forest).Evaluación: Análisis profundo mediante Matrices de Confusión, Recall y F1-Score.🛠️ Tecnologías UtilizadasPython (Pandas, NumPy)Scikit-Learn (Modelado y Preprocesamiento)Imbalanced-learn (SMOTE para balanceo)Seaborn & Matplotlib (Visualización de métricas e importancia de variables)🧠 Flujo de Trabajo (Pipeline)1. Preprocesamiento y LimpiezaSe eliminaron identificadores únicos (customerID) y se transformaron las variables categóricas a formato numérico mediante One-Hot Encoding. Para los modelos sensibles a la escala, se aplicó StandardScaler.2. Análisis de CorrelaciónSe identificaron los principales disparadores de fuga:Factores de Riesgo: Contratos mensuales, servicio de Fibra Óptica y pagos mediante cheque electrónico.Factores de Retención: Antigüedad del cliente (Meses de permanencia) y contratos a largo plazo (2 años).3. Entrenamiento y EvaluaciónSe compararon dos arquitecturas distintas para asegurar robustez:MétricaRegresión LogísticaRandom ForestExactitud~80%~79%Recall (Sensibilidad)AltoModeradoF1-ScoreEquilibradoRobustoNota: Se priorizó el Recall para minimizar los Falsos Negativos (clientes que el modelo no detecta y terminan cancelando).💡 Insights y ConclusionesEl Contrato es Clave: Los clientes con contratos "Mes a Mes" tienen una probabilidad de fuga drásticamente superior.Estrategia de Producto: Aunque la Fibra Óptica es un producto premium, presenta altas tasas de cancelación, sugiriendo una oportunidad de mejora en el servicio o ajuste de precio.Recomendación: Implementar una campaña de migración de contratos mensuales a anuales para los clientes identificados con "Alto Riesgo" por el modelo.🚀 Cómo utilizar este repositorioClona el proyecto: git clone https://github.com/tu-usuario/nombre-repo.gitInstala las dependencias: pip install -r requirements.txtEjecuta el notebook: Challenge_Telecom_X_ML.ipynb
